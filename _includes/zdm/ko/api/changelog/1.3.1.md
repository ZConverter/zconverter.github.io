
<details markdown="1">
<summary><strong>v1.3.1</strong> — Schedule 검증 개선 및 Backup Update 모드 자동 전환</summary>

**Schedule 타입 입력 개선**

- `POST /backups`, `PUT /backups/:identifier`, `POST /recoveries`, `PUT /recoveries/:identifier` — Schedule `type` 필드에 숫자(0~11) 외 문자열 입력 지원

**Backup Update 모드-스케줄 자동 전환**

- `PUT /backups/:identifier` — Smart 스케줄 ↔ Full/Increment 모드 간 변경 시 작업 모드와 스케줄을 자동으로 정합화 (Smart로 등록 시 Smart 모드 / Full·Increment로 등록 시 Full 모드 + Advanced 스케줄 초기화)

**Smart Schedule 검증 강화**

- 기존 스케줄 ID 참조 시 Smart 모드의 Basic/Advanced 정합성 검증 추가
- Smart 모드 불일치 시 4xx 응답 (에러코드 `JOB-ERROR-108`)

**Schedule 삭제 개선**

- `DELETE /schedules/:identifier` — 작업(Backup/Recovery/Replication)에서 참조 중인 스케줄 삭제 시 4xx 응답 (에러코드 `SCHEDULE-ERROR-51`)
- `DELETE /schedules/:identifier` — 응답에 `type`, `description` 필드 추가

**Backup Update 스케줄 처리 수정**

- `PUT /backups/:identifier` — 기존 스케줄 ID(숫자)로 변경 시 정상 동작
- `PUT /backups/:identifier` — 스케줄 변경 시 호환성 검증 추가

**Schedule Description 변환 수정**

- Smart Weekly / Smart Custom Monthly by Date 스케줄 설명 텍스트 정정
