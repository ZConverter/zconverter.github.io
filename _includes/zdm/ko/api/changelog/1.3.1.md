<summary><strong>v1.3.1</strong> — Schedule 검증 개선 및 Backup Update 모드 자동 전환</summary>

**Schedule 타입 입력 개선**

- `POST /backups`, `PUT /backups/:identifier`, `POST /recoveries`, `PUT /recoveries/:identifier` — Schedule `type` 필드에 숫자(0~11) 외 문자열 입력 지원

**Backup Update 모드-스케줄 자동 전환**

- `PUT /backups/:identifier` — Full/Increment 작업에 Smart 스케줄 등록 시, 작업 모드를 Smart로 자동 전환
- `PUT /backups/:identifier` — Smart 작업에 Full/Increment 스케줄 등록 시, 작업 모드를 Full로 자동 전환 + Advanced 스케줄 초기화
- `PUT /backups/:identifier` — Smart에서 Full/Increment로 모드 변경 시, 기존 스케줄(Basic/Advanced) 초기화

**Smart Schedule 검증 강화**

- 기존 스케줄 ID 참조 시 Smart 스케줄의 Basic/Advanced 모드 정합성 검증 추가
- Smart 모드 불일치 에러코드 추가 (`JOB-ERROR-108`: Schedule mode mismatch)

**Schedule 삭제 개선**

- `DELETE /schedules/:identifier` — 작업(Backup/Recovery/Replication)에서 참조 중인 스케줄 삭제 시 에러 반환 (`SCHEDULE-ERROR-51`: Schedule in use)
- `DELETE /schedules/:identifier` — 삭제 응답에 `type`, `description` 필드 추가

**Backup Update 스케줄 처리 수정**

- `PUT /backups/:identifier` — 기존 스케줄 ID(숫자)로 변경 시 정상 동작하도록 수정
- `PUT /backups/:identifier` — 스케줄 변경 시 호환성 검증(`validateJobScheduleCompatibility`) 추가

**Schedule Description 변환 수정**

- Smart Weekly(타입 7) Basic 스케줄 설명에 시간 누락 수정
- Smart Custom Monthly by Date(타입 11) Advanced 스케줄 설명의 월/날짜 순서 수정
