<summary><strong>v2.0.0</strong> — Replication 등록 API 단순화 및 GET 엔드포인트 Center 필터 추가</summary>

**Breaking Changes — Replication 등록 Body Schema 단순화**

- `POST /replications` — `sourceRepository`, `targetRepository`, `sourceServer` 필드를 단순 문자열(comma-separated)로 변경하고, `backupJobName` 필드를 `backup`으로 통합
  - `sourceRepository`: 배열 → 문자열 (예: `"39,40"`)
  - `targetRepository`: 객체 → 문자열 (예: `"41"`)
  - `sourceServer`: 배열 → 문자열 (예: `"1,server1"`)
  - `backupJobName`(배열) → `backup`(문자열, 예: `"1,job1,123"`)
- v1 호환 엔드포인트(`REPLICATION_VERSION=1` 환경)도 동일 적용

**신규 기능 — GET 엔드포인트 Center 필터 추가**

- 조회 엔드포인트에 `center` query parameter 지원 (comma-separated, ID/Name 혼용 가능)
- 적용 엔드포인트: `/backups`, `/backups/histories`, `/recoveries`, `/recoveries/histories`, `/replications`, `/replications/histories`, `/os-replications`, `/os-replications/histories`, `/servers`, `/servers/:identifier/partitions`, `/schedules`, `/licenses`, `/cloud-auth/zos`, `/cloud-auth/recovery`, `/zdms/repositories`
- 예: `?center=destconm,destconm2` 또는 `?center=9,10`

**Bug Fix — Backup Image 조회 정확성 개선**

- `POST /recoveries`, `GET /backups/images/server/:serverName` — 다중 Center 환경에서 잘못된 Center의 backup image가 매칭되거나, 다른 사용자/저장소의 동일 파일명이 잘못 매칭되던 케이스 수정

**운영 개선**

- Graceful Shutdown 단축 — 종료 응답성 향상
- `POST /recoveries` — backup image 조회 timeout이 처리 대상 파티션 수에 비례하여 자동 확장. 다중 파티션 처리 시 timeout 실패 해소

**Bug Fix — Schedule 처리 정책 정리**

- `PUT /backups/:identifier` — schedule 변경 시 동작 규칙 명문화
  - 모드 동일(F/I→F/I, Smart→Smart) + 기존 스케줄 있음 → 기존 스케줄 ID 보존하며 데이터만 갱신
  - 모드 전환(F/I↔Smart) → 신규 스케줄 등록
  - 기존 스케줄 없음 → 신규 스케줄 등록
  - 입력이 스케줄 ID(숫자) + 기존 스케줄 있음 → 입력 ID의 데이터를 기존 스케줄에 복사
- 기존 스케줄 ID 참조 시 충돌이 발생하던 문제 해결

**Bug Fix — Schedule 작업명 매핑**

- `POST /recoveries`, `PUT /recoveries/:identifier`, `PUT /backups/:identifier` — 스케줄 등록/변경 시 작업 이름이 스케줄에 매핑되지 않던 문제 수정

**UX 개선 — 에러 응답 메시지 구체화**

- 필수 필드 누락 시 응답 메시지 `"Invalid input"` → **`"<field> is required"`**
- `Internal server error.` 단독 응답 제거 — 원본 에러 메시지가 함께 포함
- `PUT /backups/:identifier` 에러 응답에 실패 단계·원인 detail 자동 포함
- Smart schedule 호환성 검증 실패 메시지에 실제 값 포함

**License 도메인 정합화 (Breaking Change 일부 포함)**

- `POST /licenses` 응답 — `copies.usage` 타입 `string("0%")` → **`number(0)`**, `dates.daysRemaining` 타입 `string("expired"|"N")` → **`number(만료=0)`** (Get 응답과 형식 통일)
- `POST /licenses` / `PUT /licenses/assign` 요청 본문 `user` 필드 — 두 엔드포인트 모두 `string | number` 허용으로 통일 (ID는 숫자, 이메일·이름은 문자열)
- `PUT /licenses/assign` 요청 본문 `center` — 빈 문자열 입력 거부 (4xx)
- `GET /licenses` 응답 — `copies.available`이 비정상 데이터에서 음수로 노출되던 것 0으로 가드
- `PUT /licenses/assign` 사전 검증 3종 추가
  - License와 Server가 동일 center에 속하는지 확인 (불일치 시 **403 CENTER_MISMATCH**)
  - 대상 Server에 이미 라이선스가 할당되어 있으면 거부 (**409 ALREADY_ASSIGNED**)
  - 한 번 할당된 server는 재할당 불가 (**409 ALREADY_ASSIGNED**)

**License 도메인 Critical Bug Fix**

- `POST /licenses/assign` — 트랜잭션 이력 기록 정합성 복구 (이력 누락 가능성 해소)
- `POST /licenses` 응답 — `category` 필드를 실제 라이선스 종류로 정확히 반환 → **`"ZDM(Backup)" / "ZDM(DR)" / "ZDM(Migration)"`** (Get/Assign 응답과 일관) **(Breaking Change)**
- `POST /licenses` 요청 `key` — 보안 강화. 영숫자 외 문자(하이픈·특수문자 포함)는 400 Bad Request로 거부됨 **(Breaking Change)**

**License 엔드포인트 동작 정상화**

- `GET /licenses/:identifier` — 단건 조회가 임의의 한 건을 반환하던 문제 수정. 정확히 해당 식별자의 라이선스 반환, 없으면 404 NOT_FOUND
- `GET /licenses/key/:key` — path param이 무시되고 전체 목록이 반환되던 문제 수정. 실제 key 기반 단건 조회로 정상 구현 (없으면 404)
- 내부 에러가 `500 Internal Server Error`로 덮이지 않고 원래 HTTP 코드(`404 NOT_FOUND` / `403 CENTER_MISMATCH` / `409 ALREADY_ASSIGNED` 등)로 정확히 전파

**License `GET /licenses` 응답 `daysRemaining` 의미 통일 (Breaking Change)**

- 기존: Get 응답은 `created → expires` 전체 유효기간 / Regist 응답은 `now → expires` 남은 일수
- 변경: 두 응답 모두 **`now → expires` 남은 일수**로 통일 (만료 시 `0`)
