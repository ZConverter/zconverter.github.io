<summary><strong>v2.0.1</strong> — Server Partition 정합성 / Recovery Windows 파티션 입력 정규화 / Job-name 일관성</summary>

v2.0.0과 사용자 가이드(엔드포인트·요청/응답 스키마) 변경 없음. 자잘한 결함 정리 모음.

**Bug Fix — Server Partition 조회 정합성**

- `GET /servers/:identifier/partitions` 등 server-partition 조회 — swap·cdrom 등 가용 용량 0인 파티션이 결과에 포함되던 케이스 수정. 모든 조회 경로에 일관 적용

**Bug Fix — Recovery Windows 파티션 입력 형식 확장**

- `POST /recoveries`, `PUT /recoveries/:identifier` — Windows 파티션 입력 매칭 범위 확장
  - 기존: `C`로 보내면 `PARTITION_NOT_FOUND`
  - 변경: `C`, `c`, `C:`, `c:` 모두 매치
- Linux 분기는 변경 없음

**Behavior Change — Job 이름 생성기 Windows/Linux 형식 통일**

- `POST /recoveries`, `POST /backups` 등 응답의 `jobName`에서 Windows 파티션 prefix 형식이 Linux와 비대칭이던 문제 정정
  - Linux: `/var/log` → `_var_log`
  - Windows (기존): `C:` → `C` (언더스코어 prefix 없음)
  - Windows (변경): `C:` → `_C` (Linux와 동일하게 언더스코어 prefix 적용)
  - 예시: `WIN-SRV01` + `C:` → `jobName` `WIN-SRV01C` → **`WIN-SRV01_C`**

**Bug Fix — 입력 누락 에러코드 정확화**

- 일부 입력 누락 케이스에서 `NO_PARTITIONS_TO_PROCESS`(400, 파티션 없음)로 부정확하게 응답되던 케이스를 누락 항목에 맞는 에러코드로 정정

**신규 에러코드**

- `JOB-ERROR-22` (HTTP 400) — 입력 누락 일반용

**운영 개선**

- Recovery / Backup Monitoring 내부 구조 단순화 — **외부 응답 시그니처·구조 변경 없음 (클라이언트 영향 없음)**
