<summary><strong>v1.3.1</strong> — Schedule 검증 강화 / 출력 형식 개선</summary>

**변경 사항 — Schedule 양식 검증 추가**

schedule 입력값을 mode에 따라 검증합니다.

| 검증 항목 | 설명 |
|-----------|------|
| smart 모드 구조 검증 | `{ basic, advanced }` 객체 형식 필수, 두 필드 모두 존재해야 함 |
| full/increment 모드 구조 검증 | `{ basic, advanced }` 형식 사용 불가 (단일 schedule 값만 허용) |
| schedule type 검증 | 숫자 0~11 또는 허용된 문자열만 유효 |
| 내부 JSON 문법 검증 | schedule 객체 또는 문자열 필드의 JSON 문법 유효성 확인 |

- 적용 커맨드: `backup regist`, `backup update`, `recovery regist`, `recovery update`

**변경 사항 — Backup regist smart 모드 schedule 필수화**

- `backup regist` 에서 `--mode smart` 지정 시 schedule 옵션(`--schedule`, `--schedule-id`, `--schedule-file`) 중 하나가 반드시 필요합니다.
- 미입력 시 에러: `smart mode requires schedule (use --schedule, --schedule-id, or --schedule-file)`

**변경 사항 — Backup regist/update `--schedule-id` 옵션 변경**

smart 모드에서 `--schedule-id` 옵션이 JSON 형식을 지원합니다.

| 항목 | 변경 전 | 변경 후 |
|------|---------|---------|
| 타입 | `number` | `string` |
| full/increment 모드 | 정수 ID | 정수 ID (동일) |
| smart 모드 | 미지원 | `'{"type":TYPE,"basic":ID,"advanced":ID}'` JSON 형식 |

**신규 커맨드 — `schedule delete`**

| 커맨드 | 설명 |
|--------|------|
| `schedule delete` | Schedule ID로 등록된 Schedule 삭제 |

```bash
zdm-cli schedule delete --id 123
```

- 삭제된 Schedule의 ID, 이름, 타입, 설명이 결과로 출력됩니다.

**변경 사항 — `schedule create` 서버 자동 등록**

- `schedule create` 실행 시 로컬 JSON 파일 저장 후 ZDM 서버에 자동으로 Schedule을 등록합니다.
- 기존에는 로컬 저장만 수행하여 별도로 `schedule regist`를 실행해야 했습니다.
- 출력에 서버 등록 결과(ID, 타입, 상태, 설명)가 함께 표시됩니다.

**변경 사항 — `--schedule` / `--schedule-file` 입력 시 basic 자동 래핑**

full/increment 모드에서 `--schedule` 또는 `--schedule-file`로 전달된 JSON에 `basic` 키가 없는 경우, `type`을 제외한 나머지 필드를 자동으로 `basic`으로 래핑합니다.

```bash
# 사용자 입력
--schedule '{"type":3,"time":"12:00"}'

# 내부 변환 후 API 전달
{"type":3,"basic":{"time":"12:00"}}
```

- 이미 `basic` 키가 있는 경우 변환하지 않습니다.
- smart 모드에서는 래핑하지 않습니다.
- 적용 커맨드: `backup regist`, `backup update`, `recovery regist`, `recovery update`

**변경 사항 — `schedule list --type` 문자열 입력 지원**

`schedule list`의 `--type` 옵션에서 숫자뿐 아니라 Schedule Type 이름 문자열도 입력할 수 있습니다.

```bash
# 숫자 입력 (기존)
zdm-cli schedule list --type 3

# 문자열 입력 (신규, 대소문자 무관)
zdm-cli schedule list --type daily
zdm-cli schedule list --type "Smart Weekly (Specific Day of the Week)"
```

**변경 사항 — Schedule type 출력 개선**

Schedule 관련 커맨드의 출력에서 type 값이 CLI에서 허용하는 정확한 문자열로 표시됩니다.

| 변경 전 | 변경 후 |
|---------|---------|
| `Smart-Weekly` | `Smart Custom (Specific Month and Date)` |
| `Minutely` | `every minute` |
| `Monthly` | `Monthly (Specific Week and Day of the Week)` |

- 적용 커맨드: `schedule create`, `schedule regist`, `schedule delete`, `schedule list`
