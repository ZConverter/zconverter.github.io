<summary><strong>v2.0.0</strong> — 멀티 ZDM(Center) 지원 / Replication V2 API / 커맨드 옵션 재설계</summary>

**주요 변경 — 멀티 ZDM(Center) 지원**

v2.0.0부터 다중 ZDM Center 환경을 지원합니다. 모든 커맨드에 `--center` 파라미터가 추가되었으며, 커맨드 유형에 따라 동작이 다릅니다.

| 커맨드 유형 | `--center` 미입력 시 | `--center` 입력 시 | 비고 |
|-------------|---------------------|-------------------|------|
| 조회 (list, history, monit) | 전체 Center 대상 조회 | 해당 Center만 필터링 | 쉼표 구분 복수 입력 가능 |
| 등록 (regist) | config `zdm.id` 자동 사용 | 지정한 Center로 등록 | 단일 값만 허용 |
| 수정 (update) | config `zdm.id` 자동 사용 | 지정한 Center로 소속 검증 | 단일 값만 허용 |
| 삭제 (delete) | config `zdm.id` 자동 사용 | 지정한 Center로 소속 검증 | 단일 값만 허용 |

- Center ID(숫자)와 Center 이름(문자열) 모두 사용 가능
- 조회 커맨드에서 복수 Center 필터링: `--center "1,centerName,3"`
- 적용 대상: backup, recovery, replication, os-replication, license, schedule, server

**주요 변경 — Replication V2 API 지원**

Replication 커맨드가 V2 API를 기본으로 사용합니다. V1 API도 설정을 통해 계속 지원됩니다.

| 항목 | V1 (기존) | V2 (신규 기본값) |
|------|-----------|-----------------|
| unit-type | `repository` | `backup`, `repository`, `server` |
| 복제 모드 | `full`, `increment` | `full`, `increment`, `sync` |
| Source Repository | 단일 값 | 쉼표 구분 복수 입력 |
| Server 지정 | 미지원 | `--server` 옵션 (ID/이름 혼합 가능) |
| Backup 작업 지정 | 미지원 | `--job` 옵션 (ID/이름 혼합 가능) |

API 버전 설정:
```bash
# V2 사용 (기본값)
zdm-cli config set --replication-version 2

# V1 사용 (레거시)
zdm-cli config set --replication-version 1
```

**변경 사항 — Replication 커맨드 옵션 재설계**

`replication regist` 옵션이 간소화되었습니다.

| 변경 전 (v1.x) | 변경 후 (v2.0.0) | 설명 |
|----------------|-----------------|------|
| `--backup-job-name` (--bjn) | `--job` (-j) | 백업 작업 ID 또는 이름, 쉼표 구분 |
| `--server-id` (--si) + `--server-name` (--sn) | `--server` (-s) | 서버 ID 또는 이름, 쉼표 구분 |
| `--source-repository-id` (--sri) | `--source-repository-id` (--sri) | 변경 없음, 쉼표 구분 복수 입력 지원 추가 |

- `--job`, `--server`, `--sri` 모두 ID와 이름을 혼합하여 입력할 수 있습니다.
- API에서 숫자는 ID로, 문자열은 이름으로 자동 구분하여 조회합니다.

```bash
# 예시: ID와 이름 혼합 입력
zdm-cli replication regist --tc destconm --ut backup --job "1,daily-backup" --tri 44
zdm-cli replication regist --tc destconm --ut server --server "91,ca-rocky810_172.25.0.48" --tri 44
zdm-cli replication regist --tc destconm --ut repository --sri "43,44" --tri 44
```

**변경 사항 — Replication API Request Body 변경**

| 필드 | 변경 전 | 변경 후 |
|------|---------|---------|
| `targetRepository` | `{ id, path }` 객체 | 문자열 (ID) |
| `sourceRepository` | 객체 배열 | 쉼표 구분 문자열 |
| `sourceServer` | 객체 배열 | 쉼표 구분 문자열 |
| `backupJobName` | 문자열 배열 | `backup` 필드로 변경, 쉼표 구분 문자열 |

**변경 사항 — Replication Source/Target Center**

Replication은 `--center` 대신 Source/Target Center를 별도로 지정합니다.

| 옵션 | 설명 | 필수 | Config fallback |
|------|------|------|-----------------|
| `--source-center` (--sc) | Source Center ID 또는 이름 | 선택 | `zdm.id` |
| `--target-center` (--tc) | Target Center ID 또는 이름 | 필수 | 없음 |

- 조회 시(`replication list`)에는 기존과 동일하게 `--center` 옵션으로 필터링

**변경 사항 — SSH 포트 옵션 (Replication)**

| 옵션 | 설명 | 기본값 |
|------|------|--------|
| `--port` | Target SSH 포트 | 22 |
| `--ip` | Target IP 주소 | - |

- `--port 0`으로 SSH 비활성화 가능

**변경 사항 — Config 신규 항목**

| 설정 키 | 설명 | 기본값 |
|---------|------|--------|
| `zdm.id` | 기본 Center ID (멀티 Center 지원) | - |
| `zdm.replication_version` | Replication API 버전 | `2` |

**변경 사항 — `license regist` 출력 보강 (API 응답 contract 변경 대응)**

API 응답 contract가 `copies.usage: string("0%")` → `number(0)`, `dates.daysRemaining: string("expired"|"N")` → `number(만료=0)` 으로 변경됨. CLI 표시는 사용자 가시성을 위해 보강:

- `copies.usage`: 값이 존재하면 `${value}%` 형식으로 출력 (예: `45` → `45%`, `0` → `0%`)
- `dates.daysRemaining`: `0` 인 경우 `expired`로 변환 출력, 양수는 그대로 일수 표시
- `0` 값 표시 누락 수정: number 필드에 사용되던 `||` 연산자를 `??`로 교체하여 `0`이 `"-"`로 잘못 표시되던 문제 해결 (총/사용/가용 라이선스 수가 0인 케이스)
