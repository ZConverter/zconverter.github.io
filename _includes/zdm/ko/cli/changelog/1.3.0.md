**신규 커맨드 — OS Replication**

| 커맨드 | 설명 |
|--------|------|
| `os-replication list` | OS Replication 목록/정보 조회 |
| `os-replication regist` | OS Replication 등록 |
| `os-replication update` | OS Replication 수정 |
| `os-replication delete` | OS Replication 삭제 |
| `os-replication monit` | OS Replication 모니터링 |
| `os-replication history` | OS Replication 실행 이력 조회 |

**신규 커맨드 — Cloud Auth**

| 커맨드 | 설명 |
|--------|------|
| `cloud-auth zos-list` | ZOS 인증 목록 조회 |
| `cloud-auth zos-regist` | ZOS 인증 등록 |
| `cloud-auth zos-delete` | ZOS 인증 삭제 |
| `cloud-auth zos-download` | ZOS 인증 키 파일 다운로드 |

| `cloud-auth recovery-delete` | Recovery 인증 삭제 |
| `cloud-auth region` | 클라우드 리전 목록 조회 |

**변경 사항 — `--schedule` 옵션 3개로 분리**

기존 `--schedule` 옵션 하나로 스케줄 ID, JSON 문자열, 파일 경로를 자동 감지하던 방식에서 용도별로 명시적으로 분리했습니다.

| 옵션 | 약어 | 설명 |
|------|------|------|
| `--schedule` | `--sc` | 스케줄 JSON 문자열 직접 입력 |
| `--schedule-id` | `--sc-id` | 기존 스케줄 ID 지정 |
| `--schedule-file` | `--sc-f` | 스케줄 JSON 파일 경로 지정 |

- 3개 옵션은 상호 배타적 (동시 사용 불가)
- Help 출력에서 `[Schedule]` 섹션 별도 분리
- 적용 커맨드: backup, recovery, replication, os-replication의 regist/update
